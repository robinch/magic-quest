## Architecture

```
User adds card
  → Scryfall autocomplete (typeahead while typing)
  → Scryfall get_card (fetch image on submit)
  → Save to wishlist_cards
  → Enqueue SearchCardWorker → searches alphaspel.se → saves listings

Periodic (every 4h via Oban cron)
  → RefreshAllWorker → enqueues SearchCardWorker per wishlist card (5s stagger)
```

## Tech Stack

- Elixir/Phoenix/LiveView
- Tailwind CSS + daisyUI (component library)
- Req (HTTP), Floki (HTML parsing), Oban (background jobs)
- Tidewave (dev-only MCP server at localhost:4000/tidewave/mcp)
- Scryfall API (card search + images)
- Alphaspel.se (availability/price via HTML scraping)

## Database Schema

### wishlist_cards
| Column | Type | Notes |
|--------|------|-------|
| user_id | FK → users | required |
| card_name | string | required |
| scryfall_id | string | from Scryfall API |

## Contexts & Key Modules

### MagicQuest.Accounts
Generated by `phx.gen.auth`. Do not modify unless needed.

### MagicQuest.Wishlists — `lib/magic_quest/wishlists.ex`
- `list_cards(user)` — preloads listings
- `add_card(user, attrs)` — creates entry, enqueues search
- `remove_card(wishlist_card)` — cascades to listings
- `update_card(wishlist_card, attrs)`

### MagicQuest.Scryfall — `lib/magic_quest/scryfall.ex`
- `autocomplete(query)` → `GET /cards/autocomplete?q=` → list of name strings
- `get_card(exact_name)` → `GET /cards/named?exact=` → card data with image_uris
- Rate limit: 100ms between requests. Image CDN (cards.scryfall.io) is unlimited.

### MagicQuest.Alphaspel — `lib/magic_quest/alphaspel.ex`
- `search(card_name)` → fetches `alphaspel.se/1978-mtg-loskort/?q=name`, follows pagination
- `parse_results(html)` → list of maps with name, set_name, price_kr, stock, condition, url
- `parse_pagination(html)` → total page count
- Rate limit: 2s between page fetches. Polite User-Agent header.

### Workers — `lib/magic_quest/workers/`
- `SearchCardWorker` — takes wishlist_card_id, searches alphaspel, replaces old listings. Oban unique.
- `RefreshAllWorker` — Oban cron every 4h, enqueues SearchCardWorker per card, 5s stagger.
- Queue config: `:search` queue, `max_concurrency: 1`

## Alphaspel.se Scraping Details

- Search URL: `https://alphaspel.se/1978-mtg-loskort/?q=card+name`
- Pagination: `?q=card+name&page=2`
- Product format: image, title ("Magic löskort: Set: Card Name"), price (kr), stock ("X i butiken" or "Slutsåld")
- Conditions in title: "(Begagnad)" = used, "(Foil)" = foil
- Product URLs: `/[set-id]-[set-slug]/[product-id]-magic-loskort-[set]-[card-slug]`
- robots.txt: allows general crawling, no crawl-delay for default user agents

## LiveView Pages

### /wishlist — WishlistLive.Index (primary page)
- Card grid with Scryfall images
- Each card: image, name, listing count, best price, last checked
- Expandable listings detail (set, price, stock, condition, alphaspel link)
- Add card form with Scryfall typeahead (300ms debounce, phx-change → autocomplete)
- "Search now" per card, "Refresh all" button
- PubSub subscription: `"user:#{user_id}:search"` for real-time updates

### / — Landing page, redirects to /wishlist if authenticated

## Typeahead Flow
1. User types → 300ms debounce → phx-change event
2. LiveView calls Scryfall.autocomplete/1
3. Render suggestions dropdown
4. Click suggestion → populate input
5. Submit → Scryfall.get_card/1 for image → save card → enqueue alphaspel search

## Testing Approach
- Alphaspel parser: HTML fixtures in `test/fixtures/`, test parse_results/parse_pagination
- Scryfall client: `Req.Test` adapter with fixture JSON responses
- Wishlists context: standard Ecto CRUD tests
- Workers: `Oban.Testing` + `Req.Test`
- LiveView: add/remove cards, typeahead, auth redirects
- Never make live HTTP requests in tests
